# Power4 Socket

> Puissance 4 en réseau — client/serveur Python avec sockets

## Stack
- Language: Python 3
- Framework: None (stdlib sockets)
- Database: None
- Package manager: pip
- Orchestration: None

## Quick Commands
```bash
pip install -r requirements.txt  # Install deps
python server/main.py            # Start server
python client/main.py            # Start client
```

## Conventions

### Code Style
- Naming: snake_case
- Max line length: 120
- Formatter: None (manual)
- Linter: None (manual)
- Docstrings: yes — Google style

### Python
- Package manager: pip (no Poetry/Pipenv unless specified)
- Virtual env: `python3 -m venv .venv` at project root
- Monorepo layout: one top-level `requirements.txt`
- Imports: absolute imports preferred over relative
- Type hints: required for public functions

### Git
- Commits in English, imperative mood, conventional format:
  `type: concise subject`
  Types: `fix:`, `feat:`, `refactor:`, `docs:`, `chore:`, `security:`
- Branch naming: `<type>/<short-desc>` (e.g. `feat/add-auth`, `fix/login-redirect`)
- PR required before merge to main: no
- Rebase, don't merge (linear history)

### Architecture
- Keep it simple — prefer native solutions over workarounds
- No dead code, no commented-out configs, no orphaned files

## Security

### Secrets
- NEVER commit: `.env`, `*.pem`, `credentials.json`, `*.key`, API keys, tokens
- Secrets live in: `.env` (gitignored) or Vaultwarden
- Scan before commit: `git diff --staged | grep -E '(password|secret|token|key|api)'`

### Dependencies
- Audit before adding: check maintenance status, known CVEs, supply chain
- Pin versions — no floating `latest` / `@latest` tags
- Prefer well-maintained, minimal-dependency libraries

### Code
- No `eval()`, no `shell=True` with user input, no unsafe deserialization
- Validate all inputs at boundaries (API, CLI, form)
- Least privilege for DB connections, API tokens, file access
- No hardcoded credentials — ever

### Network & Exposure
- No direct exposure to the internet — use Cloudflare Tunnel, VPN, or reverse proxy
- Default-deny firewall: allow only required ports
- Internal services communicate via `.ha` hostnames or 192.168.x.x, never public IPs
- `chattr +i` on critical config files where appropriate (resolv.conf, wireguard)
